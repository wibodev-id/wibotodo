## Coolify resources

### 1. Postgres database

- **Name:** `wibotodo-db`
- **Image:** `postgres:16-alpine`
- **Database:** `wibotodo`
- **User:** `wibotodo`
- **Password:** auto-generated by Coolify

The internal connection string is `postgres://wibotodo:<pw>@wibotodo-db:5432/wibotodo`.

### 2. Backend application

- **Source:** GitHub `wibodev-id/wibotodo`
- **Branch:** `main`
- **Build pack:** Dockerfile
- **Base directory:** `/` (repo root)
- **Dockerfile location:** `apps/backend/Dockerfile`
- **Port:** `3000`
- **Domain:** `https://api.todo.wibosystems.com`
- **Persistent storage:** `/app/uploads` (mount a 5 GB volume)

#### Environment variables

```
DATABASE_URL=postgres://wibotodo:<pw>@wibotodo-db:5432/wibotodo?schema=public
JWT_SECRET=<openssl rand -hex 32>
JWT_EXPIRES_IN=7d
UPLOAD_DIR=/app/uploads
MAX_UPLOAD_SIZE_MB=5
PORT=3000
CORS_ORIGIN=https://todo.wibosystems.com
SEED_ON_BOOT=true
```

> Set `SEED_ON_BOOT=false` after the first successful boot so the demo data stays
> unchanged after manual edits.

### 3. Frontend application

- **Source:** GitHub `wibodev-id/wibotodo`
- **Branch:** `main`
- **Build pack:** Dockerfile
- **Base directory:** `/` (repo root)
- **Dockerfile location:** `apps/frontend/Dockerfile`
- **Port:** `80`
- **Domain:** `https://todo.wibosystems.com`

#### API URL

The production API URL is hardcoded in `apps/frontend/src/environments/environment.prod.ts`
(`apiUrl: 'https://api.todo.wibosystems.com'`) and swapped in at build time via
`angular.json` → `fileReplacements`. No Docker build argument is needed.
